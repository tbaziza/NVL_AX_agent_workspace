# BUG-029: `iosf_sb_jem_tracker` tlm_post FAIL — `ModuleNotFoundError: No module named 'PortIdNames'`

## Symptom

Both DOA tests (`spacedoa_mobile`, `spacex_mobile`) emulate successfully ("No failing assertions found!"),
but `tlm_post` reports `iosf_sb_jem_tracker` failure with:
```
ModuleNotFoundError: No module named 'PortIdNames'
```
in `tlm_post/iosf_sb_jem_tracker/main_runner.log.gz` (post-processing replay stage).
The `iosf_sb_jem_tracker.log.gz` in the test dir is 44 bytes (nearly empty — minimal IOSF SB traffic in DOA tests).

## Triggered By

DOA test run completing emulation successfully, followed by `tlm_post` post-processing stage that invokes the IOSF sideband JEM tracker.

## Root Cause

The JEM config script for the NVL hub CDIE IOSF sideband tracker is:
  `$WORKAREA/subip/vip/sbr_network_gen/iosf_jem_tracker/nvl_hub_cdie_iosf_jem_tracker.py`

This script dynamically appends:
  `sbr_path = f"{WORKAREA}/src/codegen/{DUT}/sbr"`
to `sys.path` and then imports `PortIdNames`.

The required file `$WORKAREA/src/codegen/nvlsi7_n2p/sbr/PortIdNames.py` does not exist because
the **SBR network gen codegen flow was never run** for this workspace.
`PortIdNames.py` is generated by `subip/vip/sbr_network_gen/iosf_jem_tracker/PortIdNameGen.py`
using portid JSON files from the project's SBR network definition.

## Fix / Solution

**Proper fix (TODO)**: Run the SBR network gen codegen flow to produce `src/codegen/nvlsi7_n2p/sbr/PortIdNames.py`:
```bash
cd $WORKAREA
python3 subip/vip/sbr_network_gen/iosf_jem_tracker/PortIdNameGen.py \
  --portid_repo <path_to_nvl_portid_jsons> \
  --outfile src/codegen/nvlsi7_n2p/sbr/PortIdNames.py \
  --die CDIE
```
The portid JSON repo path needs to be obtained from the NVL SBR network gen flow.
Also needed: `SaiMap.py` for `from SaiMap import sai_map`.

**Workaround applied (2026-04-27)**:
Added `not_fail_on iosf_sb_jem_tracker` to the DOA test reglist to skip the tracker failure
without blocking the overall test verdict:
  `reglist/nvlsi7_n2p/emu/doa_pkg_ghpf_model_zse5.list` — line ~48

```
+defaults -tlm_post -not_fail_on iosf_sb_jem_tracker -tlm_post-
```
